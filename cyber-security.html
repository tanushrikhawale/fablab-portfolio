Networking – 7-Day Module

This 7-day module explored the theory and practice of computer networking — from basic concepts to diagnostic tools, and finally applying protocols in an embedded project.

Day 1 – Foundations of Networking

We began by defining the essentials of communication: sender, receiver, medium, message, and protocol. Modes of transmission (simplex, half-duplex, full-duplex) and analog vs digital signals were explained. We also studied performance factors: bandwidth, latency, reliability, and error rate.

Activity: Drew a home network diagram and traced how a smartphone communicates with a laptop over Wi-Fi.

Day 2 – Types of Networks & Data

Classified networks: PAN, LAN, MAN, and WAN. Compared client-server vs peer-to-peer models, and discussed the nature of transmitted data (structured transactions, IoT telemetry, analytics).

Activity: Identified real-world examples of each type — Bluetooth headphones (PAN), college Wi-Fi (LAN), metro city WAN connections.

Day 3 – OSI Model

The 7-layer OSI model was studied in detail: Physical, Data Link, Network, Transport, Session, Presentation, and Application. Each layer’s responsibility and common protocols (Ethernet, IP, TCP/UDP, HTTP) were discussed. Encapsulation and de-encapsulation illustrated how data travels. 

# Diagnostics
ping 8.8.8.8
traceroute google.com

Activity: Used ping and traceroute to see packet flow and intermediate hops.

Day 4 – DNS & HTTP

Learned about DNS resolving hostnames to IPs, caching, TTLs, and recursive vs authoritative servers. Explored HTTP/HTTPS request methods, status codes, and REST principles. Observed real requests in browser developer tools. 

nslookup example.com
curl -I https://example.com

Activity: Inspected DNS records and HTTP headers for caching policies.

Day 5 – DHCP & Addressing

Covered DHCP’s Discover-Offer-Request-Acknowledge process and its role in automating IP assignments. Compared static vs dynamic addressing. Reviewed real DHCP lease tables on routers. 

# Example DHCP lease
IP Address : 192.168.1.25
MAC Addr   : 00:1A:2B:3C:4D:5E
Lease Exp. : 24 hours

Activity: Configured manual static IPs and compared to DHCP-assigned addresses.

Day 6 – ARP & ICMP

Explored ARP for mapping IP to MAC addresses on a LAN. Used ICMP (ping) for diagnostics. Verified ARP tables and packet exchanges with Wireshark. 

arp -a         # view ARP table
ping 192.168.1.1
# Capture live packets
sudo wireshark

Activity: Detected a misconfigured gateway by analyzing ARP replies and ICMP failures.

Day 7 – Embedded Networking Project

Final day project: implementing DHCP, ARP, and ICMP on a Zynq FPGA board. The device successfully booted, acquired an IP via DHCP, and responded to ping requests. This reinforced how the same core protocols apply in IoT and embedded contexts.

Outcome: Demonstrated a working embedded device on the lab network, applying all learned protocols together.

✅ Learning Outcomes

  • Understood the OSI model and protocol responsibilities.
  • Practiced IP addressing, DHCP, and subnetting.
  • Gained confidence with tools: ping, traceroute, nslookup, curl, Wireshark.
  • Applied ARP and ICMP in troubleshooting.
  • Completed an embedded networking project integrating all concepts.
